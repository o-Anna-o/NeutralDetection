# Neutral Emotion Detector - Android приложение

Приложение для обнаружения и отслеживания нейтрального (спокойного) эмоционального состояния человека по кадрам видеопотока с камеры в реальном времени.

## Технологии

- **Язык**: Kotlin
- **Минимальная версия Android**: API 24 (Android 7.0)
- **Архитектура**: MVVM (Model-View-ViewModel)
- **Библиотеки**:
  - OpenCV 4.10.0 – компьютерное зрение, предобработка изображений
  - MediaPipe Face Mesh – детекция 468 лицевых landmarks
  - CameraX – работа с камерой
  - TensorFlow Lite (опционально) – нейросетевой классификатор

## Математика и алгоритмы

### 1. Геометрический анализ (OpenCV + MediaPipe)

Извлекаем 468 точек лица с помощью MediaPipe Face Mesh. Для определения «спокойствия» вычисляются следующие коэффициенты:

#### Mouth Opening Ratio (MOR)
Отношение вертикального расстояния между губами к ширине рта.

```
MOR = (lip_upper_y - lip_lower_y) / (lip_right_x - lip_left_x)
```

#### Eyebrow Position (EP)
Расстояние от бровей до центров глаз. Удивление (поднятые брови) или гнев (сведенные брови) изменяют этот параметр. Используются индексы 107, 336 (внутренние углы бровей).

#### Eye Aspect Ratio (EAR)
Степень открытости глаз.

```
EAR = (||p2-p6|| + ||p3-p5||) / (2 * ||p1-p4||)
```

### 2. Нейросетевой классификатор (TFLite)

Поверх извлеченных координат (Landmarks) запускается легковесная нейросеть (MLP или MobileNetV2-backbone), которая принимает на вход вектор нормализованных координат и выдает вероятность класса «Neutral».

## Этапы разработки и отладки

1. **Подготовка окружения**: Интеграция OpenCV SDK и MediaPipe Tasks Vision в Android Studio через Gradle.
2. **Захват видео (CameraX)**: Настройка ImageAnalysis для получения кадров в реальном времени.
3. **Предобработка (OpenCV)**:
   - Конвертация ImageProxy в Mat.
   - Применение Core.rotate для коррекции ориентации кадра.
   - Нормализация яркости (CLAHE) для стабильной работы в темноте.
4. **Анализ (MediaPipe)**: Извлечение 3D-координат.
5. **Логика детекции**: Написание функции-классификатора, проверяющей вхождение параметров в «коридор спокойствия».
6. **Отладка**: Использование физического смартфона для замера FPS (целевое значение > 25 FPS).

## Настройка и запуск

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd NeutralDetection
```

### 2. Открытие проекта в Android Studio

- Откройте папку проекта в Android Studio (версия Arctic Fox или выше).
- Дождитесь синхронизации Gradle.

### 3. Установка зависимостей

Зависимости автоматически загрузятся через Gradle. Убедитесь, что в `build.gradle.kts` указаны корректные версии:

```kotlin
implementation(libs.opencv)
implementation("com.google.mediapipe:tasks-vision:0.10.10")
```

### 4. Запуск на эмуляторе или устройстве

- Подключите Android-устройство с включенной отладкой по USB или создайте эмулятор с камерой.
- Выберите устройство в Android Studio и нажмите **Run**.

## Использование приложения

1. При первом запуске предоставьте разрешение на использование камеры.
2. Направьте фронтальную камеру на лицо.
3. На экране отобразится:
   - Статус эмоции («Нейтральное состояние», «Эмоция обнаружена», «Лицо не обнаружено»)
   - FPS (кадры в секунду)
   - Коэффициенты рта, глаз, бровей
   - Уверенность определения (прогресс-бар)
4. При нейтральном состоянии статус станет зелёным, при обнаружении эмоции – красным.

## Настройка порогов

Пороговые значения для «коридора спокойствия» настраиваются в `EmotionAnalyzer.kt`:

```kotlin
data class Thresholds(
    val mouthRatioMin: Float = 0.05f,
    val mouthRatioMax: Float = 0.25f,
    val eyeRatioMin: Float = 0.15f,
    val eyeRatioMax: Float = 0.35f,
    val eyebrowRatioMin: Float = 0.1f,
    val eyebrowRatioMax: Float = 0.4f,
    val stabilityFrames: Int = 5
)
```
