# Архитектура Android-приложения для обнаружения нейтрального эмоционального состояния

## Обзор проекта
Приложение для Android, использующее компьютерное зрение (OpenCV, MediaPipe) для обнаружения и отслеживания нейтрального (спокойного) эмоционального состояния человека в реальном времени через камеру устройства.

## Технические требования
- Минимальная версия Android: API 24 (Android 7.0)
- Язык программирования: Kotlin
- Основные библиотеки:
  - OpenCV 4.10.0 (компьютерное зрение)
  - MediaPipe Face Mesh (детекция лицевых landmarks)
  - CameraX (работа с камерой)
  - TensorFlow Lite (опционально, для нейросетевого классификатора)

## Архитектурные компоненты

### 1. Модуль захвата видео (CameraX)
- Использование `PreviewUseCase` для отображения preview
- `ImageAnalysisUseCase` для обработки кадров в реальном времени
- Обработка ориентации камеры и разрешения

### 2. Модуль обработки изображений (OpenCV)
- Конвертация `ImageProxy` в `Mat` (формат OpenCV)
- Коррекция ориентации с помощью `Core.rotate`
- Нормализация яркости (CLAHE) для улучшения детекции
- Предобработка для MediaPipe

### 3. Модуль детекции лица (MediaPipe)
- Инициализация `FaceLandmarker` с конфигурацией для 468 landmarks
- Обработка каждого кадра для получения координат лица
- Фильтрация результатов (уверенность детекции > 0.7)

### 4. Модуль анализа эмоций
#### Геометрический анализ:
- **Mouth Opening Ratio (MOR)**: `(lip_upper_y - lip_lower_y) / (lip_right_x - lip_left_x)`
- **Eyebrow Position (EP)**: расстояние от бровей до центров глаз
- **Eye Aspect Ratio (EAR)**: `(||p2-p6|| + ||p3-p5||) / (2 * ||p1-p4||)`

#### Нейросетевой классификатор (опционально):
- Вход: нормализованные координаты 468 landmarks (936 значений)
- Архитектура: MLP (3 слоя) или MobileNetV2-backbone
- Выход: вероятность класса "Neutral" (0-1)

### 5. Логика принятия решения
- Определение "коридора спокойствия" для каждого параметра
- Комбинированная оценка: если все параметры в пределах нормы → нейтральное состояние
- Пороговая обработка для устранения ложных срабатываний

### 6. Пользовательский интерфейс
- `MainActivity` с `SurfaceView` для preview камеры
- Overlay для отображения landmarks и результатов анализа
- Индикаторы состояния (нейтральное/не нейтральное)
- Статистика FPS и точность детекции

## Поток данных
1. Камера → `ImageProxy` → `ImageAnalyzer`
2. `ImageAnalyzer` → `OpenCVProcessor` (конвертация в Mat)
3. `OpenCVProcessor` → `MediaPipeFaceDetector` (получение landmarks)
4. `MediaPipeFaceDetector` → `EmotionAnalyzer` (анализ параметров)
5. `EmotionAnalyzer` → `MainActivity` (отображение результата)

## Критерии качества
- Производительность: > 25 FPS на среднем устройстве
- Точность: > 85% правильных определений нейтрального состояния
- Потребление памяти: < 150 MB
- Поддержка фронтальной и задней камеры
